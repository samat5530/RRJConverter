# RRJConverter

Конвертер валют. Это web-api, возвращает json с данными о конвертации когда на него шлют GET запрос. 
<br>Поставщик данных -  сторонний api центробанка с рейтами валют. JSON формат 
<br>Сначала проект делался как монолит, после беседы со знакомым middle-разработчиком разделил на слои (onion-архитектура)

Описание архитектуры:

<br>Слои : Domain, Integrations, Web, Database. 
<br>В домене лежат основные интерфейсы IJsonApiCurrenciesService, IRepository, ICurrencyConverter.

<br>- IJsonApiCurrenciesService - интерфейс интеграции. Реализован в интеграционном слое как интеграционный сервис, определяет правила конвертации интеграционных контрактов в доменные. 
При необходимости перейти на другой поставщик данных о курсах валют достаточно реализовать этот интерфейс новым классом и добавить класс как интеграционный сервис в IServiceCollection,
удалив в коллекции предыдущий.

<br>- IRepository - интерфейс репозитория. Реализован в слое базы данных, инкапсулирует работу базы данных. При необходимости перейти на другую базу данных или провайдер данных
также можно поступить как в случае с IJsonApiCurrenciesService. В базу данных сохраняются запрошенные пользователем операции конвертаций, хранится история конвертаций.

<br>- ICurrencyConverter - интерфейс модели конвертации. Реализован в домене. Отвечает за основную бизнес-логику конвертации.

Все обращения идут к этим интерфейсам, данные передаются как модели домена. 

Для упрощенной работы с Api чтобы не смотреть на сухие JSON-ки написал 2 HTML-страницы c JS+CSS : "домашняя страница" ~index.html и "информация" ~info.html.
<br> В домашней странице можно выполнять конвертацию. Информационная страница предоставляет более детальную информацию об Api, 
в какой форме отправлять GET-запросы и т.д.

Написал парочку unit-тестов с моками. Полностью прочёл на метаните asp.net core и C# лекции, активно использовал изучаемый материал для этого Api.
